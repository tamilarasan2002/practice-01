NO INPUT TYPE USES SUPPORTED ATTRIBUTES
1 BUTTON A push button with no default behavior displaying the value of the value attribute, empty by default. id,name,
2 CHECKBOX A check box allowing single values to be selected/deselected. id,name,value
3 COLOR A control for specifying a color; opening a color picker when active in supporting browsers. id,name
4 DATE A control for entering a date (year, month, and day, with no time). Opens a date picker or numeric wheels for year, month, day when active in supporting browsers. id,name
6 DATETIME-LOCAL A control for entering a date and time, with no time zone. Opens a date picker or numeric wheels for date- and time-components when active in supporting browsers. id,name
7 E-MAIL A field for editing an email address. Looks like a text input, but has validation parameters and relevant keyboard in supporting browsers and devices with dynamic keyboards id,name,placeholder
8 FILE A control that lets the user select a file. Use the accept attribute to define the types of files that the control can select. id,name
9 HIDDEN A control that is not displayed but whose value is submitted to the server. There is an example in the next column, but it's hidden! id,name,value
10 IMAGE A graphical submit button. Displays an image defined by the src attribute. The alt attribute displays if the image src is missing. id,name,width,height
11 MONTH A control for entering a month and year, with no time zone. id,name
12 NUMBER A control for entering a number. Displays a spinner and adds default validation when supported. Displays a numeric keypad in some devices with dynamic keypads. id,name,min,max
13 PASSWORD A single-line text field whose value is obscured. Will alert user if site is not secure. id,name,placeholder
14 RADIO A radio button, allowing a single value to be selected out of multiple choices with the same name value. id,name,value
15 RANGE A control for entering a number whose exact value is not important. Displays as a range widget defaulting to the middle value. id,name
16 RESET A button that resets the contents of the form to default values. Not recommended. id,name
17 SEARCH A single-line text field for entering search strings. Line-breaks are automatically removed from the input value. id,name
18 SUBMIT A button that submits the form. id,name
19 TEL A control for entering a telephone number. Displays a telephone keypad in some devices with dynamic keypads. id,name,placeholder
20 TEXT The default value. A single-line text field. Line-breaks are automatically removed from the input value. id,name,placeholder
21 TIME A control for entering a time value with no time zone. id,name
22 URL A field for entering a URL. Looks like a text input, but has validation parameters and relevant keyboard in supporting browsers and devices with dynamic keyboards. id,name
23 week A control for entering a date consisting of a week-year number and a week number with no time zone. id,name
